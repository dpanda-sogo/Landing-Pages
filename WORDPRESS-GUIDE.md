# WordPress Developer Guide — Sogolytics Landing Pages

This guide explains how to port a landing page from this Next.js project into WordPress so that non-technical team members can make small edits (logos, copy, images) using the WordPress editor.

---

## Overview

Each landing page (e.g. `enterprise-feedback-management`) is a **self-contained static HTML file** generated by the Next.js build. The easiest way to bring it into WordPress is via a **custom page template** — the page uses its own CSS and JS, so it bypasses the WordPress theme entirely.

---

## What you need

- WordPress with **admin access**
- FTP or cPanel file manager access to the server
- The HTML file for the page (see `out/enterprise-feedback-management.html` in this repo)
- The `_next/` assets folder (CSS and JS)

---

## Step 1 — Get the built files

Run the build locally or download the `out/` folder from the repo:

```bash
git clone https://github.com/dpanda-sogo/Landing-Pages.git
cd Landing-Pages
npm install
npm run build
```

The `out/` folder will contain:
```
out/
├── enterprise-feedback-management.html   ← the page
├── _next/                                ← CSS and JS assets
│   └── static/
│       ├── css/
│       └── chunks/
```

---

## Step 2 — Upload assets to WordPress

Upload the `_next/` folder to your WordPress server:

```
/wp-content/uploads/landing-pages/_next/
```

Via **cPanel File Manager** or **FTP**:
1. Navigate to `wp-content/uploads/`
2. Create a folder called `landing-pages`
3. Upload the entire `_next/` folder inside it

---

## Step 3 — Create a custom page template

In your WordPress theme folder (`wp-content/themes/your-theme/`), create a new file:

**`template-landing-page-efm.php`**

```php
<?php
/**
 * Template Name: EFM Landing Page
 * Template Post Type: page
 */
?>
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Enterprise Feedback Management System | Sogolytics</title>
  <meta name="description" content="Unify CX and EX feedback in one enterprise platform. NPS, VoC, sentiment analysis, and real-time dashboards — HIPAA & GDPR compliant. Request a demo.">

  <!-- Google Fonts -->
  <link rel="preconnect" href="https://fonts.googleapis.com">
  <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
  <link href="https://fonts.googleapis.com/css2?family=Epilogue:wght@400;500;600;700&display=swap" rel="stylesheet">

  <!-- Landing page CSS (from _next/static/css/) -->
  <!-- Replace HASH with the actual filename from _next/static/css/ -->
  <link rel="stylesheet" href="<?php echo content_url('uploads/landing-pages/_next/static/css/HASH.css'); ?>">
</head>
<body style="margin:0;padding:0">

  <!-- =====================================================
       EDITABLE SECTION: Logos
       Replace the LOGO 1–6 divs with <img> tags
  ====================================================== -->
  <!-- Full page content is rendered by JS below -->
  <div id="__next"></div>

  <!-- Landing page JS (from _next/static/chunks/) -->
  <!-- Replace HASH values with actual filenames -->
  <script src="<?php echo content_url('uploads/landing-pages/_next/static/chunks/webpack-HASH.js'); ?>"></script>
  <script src="<?php echo content_url('uploads/landing-pages/_next/static/chunks/main-app-HASH.js'); ?>"></script>
  <script src="<?php echo content_url('uploads/landing-pages/_next/static/chunks/HASH.js'); ?>"></script>

</body>
</html>
```

> **Note:** Open `_next/static/css/` and `_next/static/chunks/` to find the actual filenames and replace the `HASH` placeholders.

---

## Step 4 — Simpler alternative: paste raw HTML

If the PHP template approach is too complex, use this simpler method:

1. Open `out/enterprise-feedback-management.html` in a text editor
2. In WordPress admin, go to **Pages → Add New**
3. Click the **three dots (⋯)** in the top right → **Code Editor**
4. Paste the entire HTML content
5. In **Page Attributes** (right sidebar) → set **Template** to **Blank / No Header No Footer** (requires a plugin or theme support)
6. Publish

> Install the free plugin **"Blank Slate"** or **"Full Width Templates"** if your theme doesn't have a blank template option.

## Step 5 — What the developer can edit

### ✅ Easy edits (find & replace in the HTML)

| What to change | How to find it |
|---|---|
| **Customer logos** | Search `LOGO 1`, `LOGO 2` etc. Replace the `<div class="logo-slot">` with `<img src="..." alt="...">` |
| **Testimonial quote** | Search `TESTIMONIAL QUOTE` and replace the placeholder text |
| **Testimonial name/title** | Search `[Name]` and `[Title]` |
| **Company logo in testimonial** | Search `[CO. LOGO]` |
| **Stats** (500+, 96, 10M+) | Search for the numbers directly |
| **Hero image** | Search `images.unsplash.com` and replace the URL |
| **Badge images** | Search `Badge_Aicpa_Soc.png` etc. |

### ✅ Color reference (for any CSS tweaks)

| Color | Hex | Used for |
|---|---|---|
| Brand green | `#0D6E65` | Accents, section labels, check icons |
| Brand teal | `#02BECC` | Highlights, hero accent |
| Yellow | `#F5E100` | All CTA buttons |
| Dark hero bg | `#061a17` | Hero background |
| Dark footer | `#0a2c28` | Footer background |

### ✅ Adding a new logo (example)

Find this in the HTML:
```html
<div class="logo-slot">LOGO 1</div>
```

Replace with:
```html
<img src="https://your-cdn.com/client-logo.png" alt="Client Name" style="height:40px;width:auto;object-fit:contain;">
```

### ✅ Replacing the testimonial

Find:
```html
[TESTIMONIAL QUOTE — 2–3 sentences ...]
```

Replace with the actual quote text. Then find `[Name]`, `[Title]`, `[Company]`, `[CO. LOGO]` and fill those in too.

## Step 6 — Form integration

The demo request forms currently have no backend. To make them submit to your CRM or email:

**Option A — HubSpot embed:**
Replace the `<form>` section with your HubSpot form embed code.

**Option B — Add action URL:**
Find `<button type="submit"` and wrap the fields in a `<form action="YOUR_ENDPOINT" method="POST">` tag.

**Option C — Use WPForms or Gravity Forms:**
Replace the form HTML with a WordPress shortcode:
```
[wpforms id="123"]
```

## Adding a new landing page

1. Build the Next.js project → a new HTML file appears in `out/`
2. Upload the new HTML to WordPress using the same method above
3. Create a new WordPress page using the new template
